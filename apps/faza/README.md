# ФАЗА

Офлайн-абстрактная дуэль для Pocket Works. Игрок ставит один камень и затем выключает одну из трёх осей соседства: `Без ↔`, `Без ↗` или `Без ↘`.

Выключенная ось действует для обоих цветов до следующего хода. Камни вдоль неё перестают быть соседями: не объединяются в группу, не делят дыхание и не продолжают путь. Перед подтверждением игра показывает связи, разрезы, будущие захваты и дыхание новой группы.

## Победа

- снять 5 камней соперника; или
- соединить свои противоположные края и сохранить путь после ответного хода.

Синий соединяет левый и правый край. Охра соединяет верхний правый и нижний левый. Самозахват запрещён. После первого хода второй игрок может обменяться сторонами.

## Режимы

- локальная дуэль на одном экране;
- одиночная игра против четырёх характеров ИИ;
- три уровня поиска;
- серии до 1, 2 или 3 побед с чередованием первого хода.

Ход ИИ воспроизводится в три читаемых этапа: клетка, выбранная выключенная ось с объяснением, применение захватов.

## Интерфейс 1.2.0

Главное меню показывает механику на мини-поле. В партии ход всегда читается как `Клетка → Разрез → Ход`, фазы собраны в один переключатель, а поле показывает только важные изменения выбранного разреза. Подробная ревизия находится в `DESIGN-REVIEW.md`.

Геометрия поля остаётся неподвижной: интерфейс меняет только цвет, прозрачность и обводку. Для портрета, короткого портрета, альбома и широкого экрана используются отдельные сетки.

```bash
node --check apps/faza/game.js
node --check apps/faza/ai.js
node --check apps/faza/app.js
node --check apps/faza/app-v2.js
node --check apps/faza/interface-v2.js
node --check apps/faza/sw.js
node apps/faza/tests/audit.mjs
node apps/faza/tests/clarity-contract.mjs
node apps/faza/tests/stability-contract.mjs
node apps/faza/tests/interface-contract.mjs
```

Отчёт о 40 конфигурациях и более 2 600 бот-партий находится в `DESIGN-AUDIT.md` и встроен в экран «Баланс и тесты».

## Хранение и офлайн

- namespace: `pocket-works:faza`;
- cache: `faza-v1.2.0`;
- состояние серии, незавершённая партия и настройки сохраняются локально;
- все игровые и общие runtime-файлы входят в Service Worker shell.
