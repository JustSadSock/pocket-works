# СГИБ

Мобильная головоломка для Pocket Works, где игрок не двигает фишки по полю, а физически складывает сам лист.

## Основной жест

Нажми на бумагу и протяни палец от подвижной отметки к её паре. Середина движения становится линией сгиба, а направление определяет, какая половина листа поднимается.

Во время движения лист показывает объёмный предварительный сгиб:

- зелёная линия и надпись «Отпусти — сгиб готов» означают, что ход будет принят;
- красная линия означает, что жест слишком короткий или ось не пересекает лист корректно;
- неверный жест не расходует ход;
- последний завершённый сгиб можно отменить.

Обработка Pointer Events защищена от потери pointer capture, `pointercancel`, скачков координат, сворачивания приложения, поворота экрана и восстановления страницы из back-forward cache.

## Правила

- тёмная капля должна совпасть с пустым тиснёным кругом;
- две капли также взаимно гасят друг друга;
- совпавшие пары исчезают;
- лист нужно очистить за указанное число сгибов;
- направление жеста важно: складывается сторона, с которой началось движение.

В главном и паузном меню доступна встроенная инструкция «Как складывать» с учебным листом.

## Режимы

- **Кампания** — десять авторских листов, обучающих горизонтальным, вертикальным, диагональным и последовательным сгибам;
- **Бесконечная стопка** — процедурные листы, каждый из которых гарантированно решается одним точным сгибом. Лучшая серия сохраняется локально.

## Надёжность

Если применение сгиба неожиданно завершается ошибкой, партия автоматически возвращается к последнему целому состоянию вместо повреждения листа. При запуске приложение проверяет каталог кампании и несколько процедурных листов. Canvas-цикл останавливается в фоне и восстанавливается после возвращения.

## Техническая часть

Приложение использует Canvas 2D, Pointer Events, shared mobile runtime, versioned local storage, Workshop Mode, процедурный Web Audio и собственный Service Worker. После первой успешной загрузки оно полностью работает без сети.
