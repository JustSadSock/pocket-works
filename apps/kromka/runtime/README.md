# Runtime tuning notes

KROMKA runtime is split into cached text parts and joined by `app.js` at startup.

Version 1.1.0 tuning goals:

- higher opening pace and earlier procedural difficulty;
- smoother lateral response, especially during boost;
- shredder pressure that advances continuously and is pushed back by skilled drifting, boosts and checkpoints;
- visible high-speed vibration without affecting reduced-motion mode.
