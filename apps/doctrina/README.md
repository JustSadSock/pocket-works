# ДОКТРИНА

A portrait-first procedural statecraft strategy for Pocket Works. The player governs a newly formed republic through laws, doctrines, conditional technologies, faction pressure, diplomacy and front-level war.

## Primary loop

1. Read the quarterly forecast and active crisis.
2. Spend political will on a law, doctrine, research programme or diplomatic action.
3. Resolve faction and event choices.
4. End the quarter and inspect the causal chronicle.
5. Adapt the state before its own institutions, economy or army turn against it.

## Systems

- Seeded world generation with four to six regions and three neighbouring states.
- Fifteen laws with support, resistance, execution and delayed consequences.
- Twelve doctrines with incompatibilities and simulation-rule changes.
- Eighteen conditional technologies whose usefulness depends on production and institutions.
- Six political factions with influence, loyalty, wealth and demands.
- Procedural events, diplomacy, war exhaustion, occupation, collapse and state archetypes.
- Front policy controls for strategy, tempo, supply and accepted losses.

## Pocket Works integration

- Uses the shared mobile runtime and portrait safe-area contract.
- Stores the current campaign in the `pocket-works:doctrina` versioned namespace.
- Exposes Workshop diagnostics, app-owned cache clearing and a complete reset action.
- Uses the managed update script and an app-owned offline Service Worker.
- Provides a persistent return control to the Pocket Works launcher.

## Design direction

The interface is a mid-century government atlas and field headquarters dossier: ivory paper, charcoal ink, faded red stamps, olive operational marks and blue-grey technical lines. It deliberately avoids generic glass cards, neon gradients and ornamental data visualisation.
