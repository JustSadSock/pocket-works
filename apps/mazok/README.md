# МАЗОК

Полноэкранная пальцевая рисовалка для Pocket Works. Приложение открывается с локальной мастерской рисунков, прячет сложные действия в нижние панели и оставляет холсту почти весь экран.

## Основной сценарий

- один палец или стилус рисует, а инструмент заливки окрашивает выбранную область;
- два пальца перемещают и масштабируют лист;
- неподвижное удержание берёт цвет пипеткой, а удержание в конце штриха выравнивает линию, круг или прямоугольник;
- нижняя «палитра большого пальца» быстро меняет инструмент, толщину и последние цвета;
- верхняя панель даёт отмену, повтор, слои, название, галерею и экспорт;
- режим «Холст на весь экран» оставляет только лист и кнопку выхода.

Шесть инструментов — карандаш, чернила, полупрозрачный маркер, заливка, ластик и лассо. Лассо перемещает выделение одним пальцем, масштабирует и вращает двумя; отдельные кнопки копируют, удаляют и точно меняют размер. Для рисующих инструментов хранится собственная толщина, не зависящая от текущего приближения.

До пяти слоёв поддерживают видимость, прозрачность, перестановку, удаление и объединение. Цветовая панель собирает наиболее используемые цвета рисунка и рассчитывает светлый, тёмный, комплементарный и соседние оттенки.

При создании доступны два режима:

- `Лист` — ограниченная бумага с заливкой и предсказуемым размером PNG;
- `Простор` — визуально бесконечная координатная сетка для схем, карт и игровых концептов. В этом режиме заливка намеренно недоступна, потому что у области нет внешней границы.

## Сохранение и передача

Рисунки хранятся как проверяемые векторные слои схемы 2 в IndexedDB `pocket-works:mazok:drawings`. Документы схемы 1 из версий 1.x автоматически мигрируют в один слой при открытии. Настройки используют пространство `pocket-works:mazok` в localStorage. Кэш Service Worker изолирован префиксом `mazok-`; его обновление не очищает базу рисунков.

После каждого законченного действия включается отложенное автосохранение и обновляется миниатюра. До десяти автоматических версий хранятся в отдельном IndexedDB store и позволяют вернуть прежнее состояние, не теряя текущее. Галерея показывает сохранённые превью немедленно, а недостающие собирает по одному в свободное время.

Рисунок можно:

- скопировать в буфер как PNG;
- отправить через Web Share с файлом;
- загрузить как PNG при отсутствии системной отправки;
- собрать в таймлапс GIF целиком на устройстве через vendored MIT-библиотеку `gifenc`;
- переименовать, продублировать или удалить после подтверждения.

## Identity

- Slug: `mazok`
- Version: `2.0.0`
- Cache: `mazok-v2.0.0`
- Storage: `pocket-works:mazok`
- Orientation: `any`
- Runtime: Quick PWA / canvas

## Проверка

Из корня репозитория:

```bash
node --check apps/mazok/app.js
node --check apps/mazok/drawing-core.js
node --check apps/mazok/drawing-db.js
node --check apps/mazok/timelapse.js
node apps/mazok/test.mjs
node apps/mazok/dom-test.mjs
npm run validate:configs
npm run validate:mobile
npm run registry:check
npm run deploy:site
```

Отдельно проверяются мобильные размеры, рисование Pointer Events, удержания, распознавание фигур, лассо, слои, undo/redo, pinch-zoom, обе модели холста, миграция схемы 1, версии IndexedDB, GIF-кодирование, галерея, экспорт, offline reload и отсутствие горизонтального переполнения.
