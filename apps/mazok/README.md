# МАЗОК

Полноэкранная пальцевая рисовалка для Pocket Works. Приложение открывается с локальной мастерской рисунков, прячет сложные действия в нижние панели и оставляет холсту почти весь экран.

## Основной сценарий

- один палец или стилус рисует, а инструмент заливки окрашивает выбранную область;
- два пальца перемещают и масштабируют лист;
- нижняя «палитра большого пальца» быстро меняет инструмент, толщину и последние цвета;
- верхняя панель даёт отмену, повтор, название, галерею и экспорт;
- режим «Холст на весь экран» оставляет только лист и кнопку выхода.

Пять инструментов — карандаш, чернила, полупрозрачный маркер, заливка и ластик. Для рисующих инструментов хранится собственная толщина, не зависящая от текущего приближения. Доступны системный выбор цвета и четыре цвета бумаги.

## Сохранение и передача

Рисунки хранятся как проверяемые векторные мазки в IndexedDB `pocket-works:mazok:drawings`. Настройки и ID последнего рисунка используют пространство `pocket-works:mazok` в localStorage. Кэш Service Worker изолирован префиксом `mazok-`; его обновление не очищает базу рисунков.

После каждого законченного мазка включается отложенное автосохранение и обновляется миниатюра. Галерея показывает уже сохранённые превью немедленно, а недостающие собирает по одному в свободное время и сразу отменяет эту работу при открытии рисунка. Перед открытием галереи и при уходе приложения в фон выполняется дополнительная запись. Рисунок можно:

- скопировать в буфер как PNG;
- отправить через Web Share с файлом;
- загрузить как PNG при отсутствии системной отправки;
- переименовать, продублировать или удалить после подтверждения.

## Identity

- Slug: `mazok`
- Version: `1.2.1`
- Cache: `mazok-v1.2.1`
- Storage: `pocket-works:mazok`
- Orientation: `any`
- Runtime: Quick PWA / canvas

## Проверка

Из корня репозитория:

```bash
node --check apps/mazok/app.js
node --check apps/mazok/drawing-core.js
node --check apps/mazok/drawing-db.js
npm run validate:configs
npm run validate:mobile
npm run registry:check
npm run deploy:site
```

Отдельно проверяются мобильные размеры, рисование Pointer Events, отмена/повтор, pinch-zoom, перезагрузка с восстановлением IndexedDB, галерея, копирование/отправка/загрузка, offline reload и отсутствие горизонтального переполнения.
